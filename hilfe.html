Grundprinzip

Die Tools erzeugen entweder Microsoft 365‑Objekte (Gruppen/Teams) über Microsoft Graph oder sie erzeugen PowerShell‑Skripte / .cmd‑Pakete, die Sie lokal ausführen. Die “Stammdaten” (Domain, Fächer, Lehrkräfte, Klassen, Schüler) kommen aus den Schul‑Grundeinstellungen.

Datenhaltung: Es gibt keinen Server. Zwischenstände und Stammdaten werden lokal im Browser gespeichert (LocalStorage/SessionStorage). Viele Tools haben zusätzlich Export/Import (JSON/CSV), um zwischen PCs/Browsers zu wechseln.

Voraussetzungen (Graph / Berechtigungen)

Wenn ein Tool “Online (Microsoft Graph)” anbietet, erfolgt die Anmeldung über MSAL im Browser. Dafür braucht die App‑Registrierung in Entra passende delegierte Berechtigungen (und meist Administratorzustimmung).

Wichtig: Die “Berechtigung” ist immer eine Kombination aus (a) in Entra erteilten Scopes und (b) den Rechten des angemeldeten Kontos (z. B. Gruppen verwalten, Teams erstellen, Directory schreiben).

Datenschutz

Es gibt keinen Server. Importierte Dateien und Einstellungen werden nur in diesem Browser verarbeitet; optional können Sie den Zwischenstand pro Modus (Kursteams, Jahrgangsgruppen, ARGEs) getrennt lokal speichern.

Module / Werkzeuge (Mini‑Anleitungen)

Schul‑Grundeinstellungen Basisdaten

Was macht’s? Speichert die gemeinsame Basis: Domain, Fächer, Lehrer, optional Schüler und Klassen. Diese Daten werden von Kursteams, Jahrgang und ARGE übernommen bzw. teilweise zurück‑synchronisiert.

Mini‑Anleitung
  1. Domain ohne @ eintragen (z. B. ms365.schule).
  2. Listen füllen: per Copy & Paste oder CSV/XLSX‑Import (Vorlagen gibt’s direkt im Tool).
  3. Mit Speichern sichern (lokal im Browser). Optional Export/Import (JSON) zum Übertragen.
Tipp: Viele Tools übernehmen automatisch Daten, wenn das jeweilige Eingabefeld leer ist (z. B. Jahrgang übernimmt Klassen, ARGE übernimmt Fächer).

Kursteams Wizard

Was macht’s? Aus Unterrichtszeilen (Lehrer, Fach, Klasse, optional Gruppe) werden Team‑Entwürfe erzeugt: Team‑Name, Gruppenmail (Nickname), Besitzer‑UPN/E‑Mail. Anlage erfolgt wahlweise direkt über Graph oder per CSV/PowerShell.

Mini‑Anleitung (Hauptweg)
  1. Daten importieren: Copy & Paste (3 Spalten) oder CSV/XLSX.
  2. Bereinigen: unnötige Fächer ausschließen, Duplikate entfernen, klassenübergreifende Teams prüfen.
  3. Lehrer zuordnen: Kürzel → E‑Mail (kommt aus der Schul‑Liste; fehlende werden aus Kürzel@Domain generiert).
  4. Team‑Namen generieren: Baukasten (Drag & Drop), dann validieren.
  5. Anlegen: Online per Graph (Login → “Kursteams jetzt anlegen”) oder Schritt “CSV/PowerShell” nutzen.
Optional: Schüler als Mitglieder hinzufügen (separater Schritt). Format z. B. Klasse;UPN oder Klasse;Gruppe;UPN.

Jahrgangsgruppen Gruppen/Teams

Was macht’s? Erstellt Microsoft 365‑Gruppen (Unified) je Klasse mit Mail‑Nickname‑Schema <Präfix><Jahr>-<Kürzel> (z. B. jg2030-AK). Optional wird auch Teams “bereitgestellt”. Ausgeführt wird über generiertes PowerShell/.cmd (und je nach Build auch online).

Mini‑Anleitung
  1. Klassenliste einfügen (z. B. 1AK;2030;Klasse 1A‑HAK oder nur 1AK).
  2. Besitzer (UPN) pro Gruppe setzen.
  3. Mitglieder optional: pro Klasse eine UPN‑Liste (eine Zeile pro Person).
  4. Einstellungen: primäre SMTP per Exchange setzen und/oder Teams erstellen.
  5. Ausführen: Script kopieren oder Jahrgangsgruppen‑Anlage.cmd herunterladen.

ARGEs Gruppen/Teams

Was macht’s? Erstellt Microsoft 365‑Gruppen für Arbeitsgemeinschaften aus einer einfachen Liste (eine Zeile pro Thema/Fach). Daraus werden ARGE …‑Anzeigenamen und Mail‑Nicknames (z. B. arge-de) generiert. Optional: Teams anbinden, SMTP per Exchange setzen, Admin zusätzlich als Besitzer.

Mini‑Anleitung
  1. In Schritt 1 je Zeile ein Thema einfügen (z. B. Deutsch, Robotics). Optional auch Anzeigename;MailNickname.
  2. Vorschau prüfen und bei Bedarf per Doppelklick Zellen korrigieren.
  3. Besitzer (UPN) eintragen, optional Mitgliederlisten ergänzen.
  4. Einstellungen wählen (Exchange‑SMTP/Teams/Admin‑Owner) und dann online ausführen oder Script/.cmd nutzen.

Schüler:innen‑ & Lehrer:innen‑Gruppen Mitglieder synchronisieren

Was macht’s? Legt zwei Unified‑Gruppen (typisch schueler und lehrer) an oder nutzt bestehende Gruppen, und fügt Mitglieder aus den Schul‑Listen hinzu (Duplikate werden übersprungen). Optional kann die Gruppe auch als Team erstellt werden.

Mini‑Anleitung
  1. In den Schul‑Grundeinstellungen Schüler/Lehrer mit gültigen E‑Mails pflegen.
  2. Im Tool anmelden, Gruppen “neu” anlegen oder “bestehend” auswählen (GUID / Suche).
  3. Mit „Liste → Gruppe“ die Mitgliedschaften synchronisieren.
  4. Fehlende Personen bei Bedarf manuell ergänzen (eine Zeile pro UPN/E‑Mail).
Hinweis: Entfernen von Mitgliedern macht dieses Tool nicht – dafür eignet sich die Gruppenübersicht oder das Entra‑Portal.

MS365‑Gruppen (Übersicht) Lesen & Bearbeiten

Was macht’s? Listet Unified‑Gruppen im Mandanten (inkl. Teams) mit Besitzern, Mitgliederanzahl und Erstellungsdatum. Optional können Besitzer/Mitglieder gesucht und hinzugefügt werden.

Mini‑Anleitung
  1. Anmelden → Gruppen laden.
  2. Optional filterbar (nur Teams / Textfilter / Besitzerfilter) und als CSV exportierbar.
  3. Zum Bearbeiten: Gruppe in der Tabelle auswählen → Benutzer suchen → als Besitzer/Mitglied hinzufügen.
Grenze: Die Namensliste zeigt aus Performance‑Gründen nur eine erste Seite (max. 40 Namen sichtbar), die Mitgliederzahl kommt separat über …/members/$count.

Teams archivieren Graph‑Aktion

Was macht’s? Archiviert ein Microsoft Team (oder hebt die Archivierung auf). Die Team‑ID ist die GUID der zugehörigen Unified‑Gruppe. Optional kann die SharePoint‑Teamwebsite für Mitglieder schreibgeschützt gesetzt werden.

Mini‑Anleitung
  1. Team‑/Gruppen‑ID (GUID) einfügen oder per Anzeigename/E‑Mail suchen.
  2. Anmelden → Team archivieren oder Archivierung aufheben.
  3. Protokoll unten prüfen (asynchrone Operation kann etwas dauern).

Klassen umbenennen Schuljahreswechsel

Was macht’s? Liest Unified‑Gruppen und berechnet eine Vorschau für eine Umbenennungsregel (typisch: Klasse 1A → Klasse 2A). Ausgeführt wird dann ein PATCH /groups auf displayName. Mail‑Nickname und Gruppen‑E‑Mail bleiben unverändert.

Mini‑Anleitung
  1. Anmelden → Gruppen laden (optional “nur Teams” / Textfilter).
  2. Regel festlegen: Präfix (z. B. Klasse) und optional Bulk‑Ersetzungen.
  3. Vorschau berechnen und kontrollieren → erst dann ausführen.
Achtung: Das ist eine echte Änderung im Mandanten. Bitte die Vorschau vollständig prüfen, bevor “Jetzt umbenennen” geklickt wird.

Gruppenerstellung einschränken Directory Setting

Was macht’s? Setzt die Entra‑Directory‑Einstellung Group.Unified, sodass nur Mitglieder einer Sicherheitsgruppe M365‑Gruppen/Teams erstellen dürfen (EnableGroupCreation=false, GroupCreationAllowedGroupId=<GUID>). Kann auch wieder entfernt werden.

Mini‑Anleitung
  1. Sicherheitsgruppe anlegen (oder vorhandene GUID eintragen / suchen).
  2. Mitglieder dieser Sicherheitsgruppe im Entra‑Portal verwalten.
  3. Im Tool anmelden → “Einschränkung anwenden” oder “Status lesen” / “Einstellung entfernen”.
Berechtigungen: benötigt u. a. Directory.ReadWrite.All (delegiert) und Admin‑Zustimmung.

Windows blockiert heruntergeladene .cmd?

Das ist normal (Internetsicherheit / Mark of the Web). Lösung: Rechtsklick auf die Datei → Eigenschaften → unten Zulassen aktivieren → OK. Danach erneut doppelklicken.

Alternativ in PowerShell:

Unblock-File -LiteralPath '…\Downloads\…-Anlage.cmd'
